# ENGINE.JS — COMPLETE DESCRIPTION OF BEHAVIOR

The JS is an IIFE that runs on DOMContentLoaded:

## 0. Auto-Wrap Section Content (runs first)
- Wraps ALL children of every `.section-content` into `<div class="section-content-inner">`
- Required for CSS Grid `0fr` collapse to work (zero height, no blank space)
- Skips if wrapper already exists
- **You do NOT add this wrapper in HTML**

## 1. Toast System
- `showToast(msg, icon)` — 2.5s notification

## 2. Dark Mode Toggle
- localStorage + system preference fallback
- `#themeToggle` and `#drawerThemeToggle` trigger theme change

## 3. Mobile Drawer
- `#menuBtn` opens, `#drawerClose` and `#navOverlay` close
- Locks body scroll when open
- Drawer link clicks: close drawer → force `.visible` → **auto-expand if collapsed** → smooth scroll with 80px offset

## 4. Desktop Nav Smooth Scroll
- Same as drawer: force `.visible` → auto-expand if collapsed → smooth scroll

## 5. Scroll Handler (rAF throttled, passive)
- Progress bar, navbar shadow, FAB visibility, active nav link highlighting

## 6. FAB Buttons
- Scroll to top, print, save with visual feedback

## 7. Collapsible Sections (CSS Grid — NO maxHeight)
- Click `.section-title` toggles `.collapsed` on `.section-content` and `.collapse-icon`
- Auto-wrapped `.section-content-inner` ensures zero blank space
- Pure CSS animation: `grid-template-rows 1fr ↔ 0fr`

## 8. Exercise Answer Toggles
- Hides all `.answer-reveal` initially
- Clones buttons to remove stale listeners
- Toggles display and updates button text

## 9. Scroll Reveal (IntersectionObserver + Fallbacks)
- Observer: threshold 0.02, rootMargin "50px 0px"
- Fallback `forceRevealVisible()` at 100ms, 500ms, 1500ms + scroll-end + resize
- Ensures fast scrolling never leaves sections invisible

## 10. Keyboard Shortcuts
- `Ctrl+P` print, `Ctrl+S` save, `Escape` close drawer

For complex math, add in `<head>`:

```html
<script>
MathJax = {
    tex: {
        inlineMath: [['\\(', '\\)']],
        displayMath: [['\\[', '\\]']],
    },
    options: {
        skipHtmlTags: ['script', 'noscript', 'style', 'textarea', 'pre']
    }
};
</script>
<script src="https://cdn.jsdelivr.net/npm/mathjax@3/es5/tex-mml-chtml.js" async></script>
```

### When to use what:

**Simple HTML fractions** for basic formulas like R=V/I:
```html
<div class="formula">
    R = <span class="fraction"><span class="num">V</span><span class="den">I</span></span>
</div>
```

Unit CSS for fractions:
```css
.fraction { display:inline-flex; flex-direction:column; align-items:center; vertical-align:middle; margin:0 4px; text-align:center; }
.fraction .num { border-bottom:2px solid currentColor; padding:0 6px 3px; line-height:1.2; }
.fraction .den { padding:3px 6px 0; line-height:1.2; }
```

**Use MathJax** for: integrals, summations, matrices, nested fractions, roots, limits, derivatives, piecewise functions, aligned equations.

MathJax inline: `\( E = mc^2 \)`
MathJax block:
```html
<div class="formula">
    \[ R = \frac{\rho \cdot l}{A} \]
</div>
```

**Simple HTML entities** (no MathJax needed): `α` `β` `Ω` `ρ` `ω` `π` `μ` `σ` `θ` `λ` `Δ` `φ` `R<sub>eq</sub>` `10⁻³` `→` `≤` `≥` `×` `±` `√` `∞`

Include MathJax ONLY if the unit actually has complex formulas.
