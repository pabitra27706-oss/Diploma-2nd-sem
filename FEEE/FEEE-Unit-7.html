Unit Overview

What This Unit Covers

This unit introduces Digital Electronics, which is the foundation of computers, calculators, mobile phones, digital clocks, PLCs, microcontrollers, and all modern digital systems.

In digital electronics, signals are represented by only two states:

  • 0 → LOW → False → OFF
  • 1 → HIGH → True → ON

You will learn:

  • Boolean algebra rules and laws
  • Logic gates and their truth tables
  • Universal gates (NAND, NOR)
  • De-Morgan’s theorem, SOP, POS, K-Map simplification
  • Basic idea of flip-flops as memory elements
  • Ripple and synchronous up-down counters
  • Introduction to TTL logic family
Why This Unit Matters
  • Computer hardware is built from logic gates
  • Memory circuits use flip-flops
  • Counters are used in digital clocks, frequency meters, event counters
  • Boolean algebra helps simplify circuits and reduce hardware cost
  • TTL ICs are basic digital building blocks used in labs and industry

7.1 Boolean Algebra — Rules and Laws

What is Boolean Algebra? 🔥⭐

Definition: Boolean Algebra

Boolean Algebra is a branch of algebra used for the analysis and simplification of digital logic circuits. It deals with only two binary values: 0 and 1. It was developed by George Boole.

Why Boolean Algebra is Needed
  • It helps to represent digital circuits mathematically
  • It helps to simplify logic expressions
  • Simpler expression = fewer gates = lower cost = less power = more reliable circuit
  • It forms the basis of computer logic design

Basic Symbols and Operations

Operation Symbol Read As Equivalent Logic Gate
AND A · B or AB A AND B AND gate
OR A + B A OR B OR gate
NOT A̅ or A' NOT A / complement of A NOT gate
Very Important Difference

In Boolean algebra:

  • + means OR, NOT arithmetic addition
  • · means AND, NOT arithmetic multiplication
  • 1 + 1 = 1 in Boolean algebra, not 2
  • 1 · 1 = 1, but 1 · 0 = 0

Basic Rules and Laws of Boolean Algebra 🔥⭐

Must-Learn Laws
  1. OR Law:
    A + 0 = A
    A + 1 = 1
  2. AND Law:
    A · 1 = A
    A · 0 = 0
  3. Idempotent Law:
    A + A = A
    A · A = A
  4. Complement Law:
    A + A̅ = 1
    A · A̅ = 0
  5. Involution Law:
    (A̅)̅ = A
  6. Commutative Law:
    A + B = B + A
    A · B = B · A
  7. Associative Law:
    (A + B) + C = A + (B + C)
    (A · B) · C = A · (B · C)
  8. Distributive Law:
    A(B + C) = AB + AC
    A + BC = (A + B)(A + C)
  9. Absorption Law:
    A + AB = A
    A(A + B) = A
  10. Important Reduction Laws:
    AB + A B̅ = A
    (A + B)(A + B̅) = A
🔥 Most Important Laws for Exam:
  • A + 0 = A
  • A · 1 = A
  • A + A̅ = 1
  • A · A̅ = 0
  • A + AB = A
  • A(B + C) = AB + AC
  • AB + A B̅ = A

Simple Boolean Simplification Examples

Example 7.1

Simplify: A + A̅B

Solution
Use distributive form:
A + A̅B = (A + A̅)(A + B)
Since A + A̅ = 1:
= 1 · (A + B)
= A + B
Example 7.2

Simplify: AB + AB̅

Solution
Take A common:
AB + AB̅ = A(B + B̅)
Since B + B̅ = 1:
= A · 1
= A

7.1 Basic Logic Gates and Truth Tables

What is a Logic Gate? 🔥⭐

Definition: Logic Gate

A Logic Gate is an electronic circuit that performs a logical operation on one or more binary inputs and produces a single binary output. Logic gates are the basic building blocks of digital systems.

Why Logic Gates Are Called Building Blocks

Just like bricks are used to build houses, logic gates are used to build:

  • Adders and subtractors
  • Multiplexers and decoders
  • Flip-flops and counters
  • Memories
  • Microprocessors and computers

Universal Logic Gates 🔥⭐

Definition: Universal Gate

A Universal Gate is a logic gate that can be used to implement all other basic logic gates (AND, OR, NOT and even complex logic functions). The two universal gates are:

  • NAND gate
  • NOR gate

Why NAND and NOR are called Universal?

Universal Property

Using only NAND gates, we can build:

  • NOT gate
  • AND gate
  • OR gate
  • Any complex digital circuit

Similarly, using only NOR gates, we can build all other gates.

Realization Using NAND Gates

NOT using NAND

Y = (A · A)̅ = A̅

Connect both inputs of NAND together.

AND using NAND

Y = ((A · B)̅)̅ = A · B

Use one NAND followed by a NAND-as-NOT.

OR using NAND

Y = (A̅ · B̅)̅ = A + B

Using De-Morgan’s theorem.

7.2 De-Morgan's Theorem

What is De-Morgan's Theorem? 🔥⭐

Definition: De-Morgan's Theorem

De-Morgan's theorem is a very important Boolean algebra theorem used to simplify logic expressions and to convert AND operations into OR operations and vice versa by complementing each variable. It is especially useful in implementing logic circuits using NAND and NOR gates.

The Two De-Morgan Theorems
  1. First Theorem:
    (A · B)̅ = A̅ + B̅

    Meaning: The complement of a product is equal to the sum of the complements.

  2. Second Theorem:
    (A + B)̅ = A̅ · B̅

    Meaning: The complement of a sum is equal to the product of the complements.

🔥 Easy Memory Rule:

When you remove the bar (complement) from a bracket:

  • Change AND to OR
  • Change OR to AND
  • Complement each variable

In short: "Break the bar → change the sign → complement each term"

Worked Examples

Example 7.3

Apply De-Morgan's theorem to simplify: (A + B + C)̅

Solution
Use second theorem:
(A + B + C)̅ = A̅ · B̅ · C̅
Answer: A̅B̅C̅
Example 7.4

Apply De-Morgan's theorem to simplify: (AB + C)̅

Solution
Treat AB as one term:
(AB + C)̅ = (AB)̅ · C̅
Now apply theorem to (AB)̅:
(AB)̅ = A̅ + B̅
(AB + C)̅ = (A̅ + B̅)C̅

7.2 SOP, POS, Minterm, Maxterm and K-Map

Minterm and Maxterm 🔥⭐

Definition: Minterm

A minterm is a standard product term in which all variables appear exactly once, either in complemented or uncomplemented form. A minterm gives output 1 for only one combination of input variables.

Definition: Maxterm

A maxterm is a standard sum term in which all variables appear exactly once, either in complemented or uncomplemented form. A maxterm gives output 0 for only one combination of input variables.

For Two Variables A and B

A B Minterm Symbol Maxterm Symbol
0 0 A̅B̅ m0 A + B M0
0 1 A̅B m1 A + B̅ M1
1 0 AB̅ m2 A̅ + B M2
1 1 AB m3 A̅ + B̅ M3
⭐ Quick Rule:
  • For minterm: 0 means complemented, 1 means normal
  • For maxterm: 0 means normal, 1 means complemented

SOP and POS Forms 🔥⭐

Definition: SOP (Sum of Products)

SOP means a Boolean expression written as the sum (OR) of product (AND) terms. Example:

F = A̅B + AB̅ + AB

Each term like A̅B or AB̅ is a product term, and they are added (ORed) together.

Definition: POS (Product of Sums)

POS means a Boolean expression written as the product (AND) of sum (OR) terms. Example:

F = (A + B)(A̅ + B)(A + B̅)

Each bracket is a sum term, and the brackets are multiplied (ANDed).

Form Full Name Built from Output based on
SOP Sum of Products Minterms Rows where output = 1
POS Product of Sums Maxterms Rows where output = 0

Canonical SOP and POS

Standard Representation

If a function is expressed in terms of minterms:

F(A,B,C) = Σm(1,3,5,7)

This means output is 1 for minterms 1, 3, 5, and 7.

If a function is expressed in terms of maxterms:

F(A,B,C) = ΠM(0,2,4,6)

This means output is 0 for maxterms 0, 2, 4, and 6.

Karnaugh Map (K-Map) 🔥⭐

Definition: Karnaugh Map

A Karnaugh Map (K-Map) is a graphical method used to simplify Boolean expressions. It arranges minterms or maxterms in a special grid so that adjacent cells differ in only one variable. By grouping adjacent 1s (for SOP) or 0s (for POS), we can obtain a simplified expression.

Why K-Map is Useful
  • Much easier than lengthy Boolean algebra for 2, 3, and 4 variable functions
  • Helps get the minimum expression
  • Reduces number of logic gates required
  • Very common in diploma exams

Important K-Map Rules

Rules for Grouping
  1. Group only adjacent 1s (for SOP simplification)
  2. Groups can contain 1, 2, 4, 8... cells (power of 2 only)
  3. Make the largest possible groups first
  4. A cell can be used in more than one group if needed
  5. Corner cells are also adjacent
  6. Top and bottom cells can be adjacent, left and right cells can be adjacent
  7. Take minimum number of groups to cover all 1s

K-Map Solved Examples

Example 7.5

Simplify the Boolean function using Boolean algebra: F = A̅B + AB

Solution
Take B common:
F = B(A̅ + A)
Since A̅ + A = 1:
F = B · 1
F = B
Example 7.6

Simplify using K-Map: F(A,B) = Σm(1,3)

Solution
For two variables:
  • m1 = A̅B
  • m3 = AB
In 2-variable K-map, m1 and m3 are adjacent vertically.
Common variable in both cells is B = 1. A changes, so A is eliminated.
F = B
Example 7.7

Simplify using K-Map: F(A,B,C) = Σm(1,3,5,7)

Solution
Place 1s in minterms 1, 3, 5, 7 on the 3-variable K-map.
These four cells form one group of 4.
In all these minterms, the common variable is C = 1. A and B change.
F = C
Common Mistakes in K-Map
  • Grouping diagonal cells — wrong
  • Making groups of 3 or 5 — wrong
  • Ignoring corner adjacency
  • Not taking the largest possible groups first

7.3 Flip-Flops as Storage Elements

What is a Flip-Flop? 🔥⭐

Definition: Flip-Flop

A Flip-Flop is a bistable multivibrator — a digital circuit that has two stable states and can store one bit of binary information (0 or 1). It is used as a basic memory element in digital electronics.

Why Flip-Flops Are Important
  • They store binary data
  • They are used in registers, counters, memory units, and sequential circuits
  • Unlike combinational circuits, flip-flops have memory of previous state
⭐ Key Difference:
  • Combinational circuit: Output depends only on present input
  • Sequential circuit: Output depends on present input + previous state

Flip-flops belong to sequential circuits.

Basic Types of Flip-Flops

SR Flip-Flop

S = Set, R = Reset

Basic latch circuit used to store 1 or 0.

JK Flip-Flop

Improved version of SR flip-flop.

No invalid state like SR flip-flop.

D Flip-Flop

D = Data/Delay

Stores the input bit directly.

T Flip-Flop

T = Toggle

Changes state on each clock pulse when T = 1.

SR Flip-Flop (Basic Concept)

SR Flip-Flop Working

SR flip-flop can be built using two cross-coupled NOR gates or NAND gates.

  • S = 1, R = 0 → Set → Q = 1
  • S = 0, R = 1 → Reset → Q = 0
  • S = 0, R = 0 → No change / memory condition
  • S = 1, R = 1 → Invalid condition (for NOR latch)

7.4 Counters — Ripple and Synchronous Up-Down Counter

What is a Counter? 🔥⭐

Definition: Counter

A Counter is a sequential digital circuit that counts the number of clock pulses applied to it. It is made of flip-flops and changes its state in a definite sequence. The output represents the count in binary form.

Applications of Counters
  • Digital clocks and timers
  • Frequency counters
  • Event counting
  • Sequence generation
  • Division of frequency
  • Control circuits and automation

Ripple Counter (Asynchronous Counter) 🔥

Definition: Ripple Counter

A Ripple Counter is an asynchronous counter in which the output of one flip-flop is used as the clock input of the next flip-flop. The clock pulses do not reach all flip-flops at the same time, so the changes ripple from one stage to the next.

Working of Ripple Counter
  1. The first flip-flop receives the external clock pulse
  2. Its output toggles on each clock pulse
  3. This output acts as the clock for the second flip-flop
  4. The second output acts as clock for the third flip-flop, and so on
  5. Thus counting happens in a chain manner — like a ripple

Important Point: Since each flip-flop waits for the previous one, ripple counter is slower.

Modulus of a Counter

Counter Modulus Formula
MOD = 2n

Where n = number of flip-flops

  • 2 flip-flops → MOD-4 counter
  • 3 flip-flops → MOD-8 counter
  • 4 flip-flops → MOD-16 counter

Synchronous Counter 🔥⭐

Definition: Synchronous Counter

A Synchronous Counter is a counter in which all flip-flops receive the clock pulse at the same time. Therefore, all flip-flops can change state simultaneously. This makes synchronous counters faster and more accurate than ripple counters.

Why Synchronous Counter is Better
  • All flip-flops are triggered simultaneously
  • No ripple delay
  • Faster operation
  • Used in high-speed digital systems

7.5 Introduction to TTL Logic Gates

What is TTL? 🔥

Definition: TTL

TTL stands for Transistor-Transistor Logic. It is a digital logic family in which both the logic function and amplification are performed using bipolar junction transistors (BJTs).

Why TTL is Important
  • Very popular digital IC family
  • Used in basic laboratory experiments and classic digital systems
  • Common TTL IC series: 74xx family
  • Fast switching compared to older RTL and DTL families

Important Definitions

1. Digital Electronics

Digital electronics is the branch of electronics that deals with circuits operating on discrete binary signals, mainly 0 and 1.

2. Boolean Algebra

A mathematical system used to represent and simplify logic expressions involving only two values: 0 and 1.

3. Logic Gate

An electronic circuit that performs a logical operation on one or more binary inputs and gives one binary output.

4. Universal Gate

A logic gate that can be used to implement all other logic gates. NAND and NOR are universal gates.

5. Minterm

A standard product term containing all variables exactly once, either complemented or not. It corresponds to one row of truth table where output is 1.

6. Maxterm

A standard sum term containing all variables exactly once. It corresponds to one row of truth table where output is 0.

7. SOP

Sum of Products — a Boolean expression written as OR of AND terms.

8. POS

Product of Sums — a Boolean expression written as AND of OR terms.

9. Karnaugh Map (K-Map)

A graphical technique used for simplifying Boolean expressions by grouping adjacent 1s or 0s.

10. Flip-Flop

A bistable sequential circuit that can store one bit of information.

11. Counter

A sequential circuit that counts clock pulses and changes states in a fixed sequence.

12. Ripple Counter

An asynchronous counter where the output of one flip-flop drives the clock of the next flip-flop.

13. Synchronous Counter

A counter in which all flip-flops receive the clock pulse simultaneously.

14. Up-Down Counter

A counter that can count in increasing or decreasing order depending on control input.

15. TTL

Transistor-Transistor Logic — a digital logic family based on bipolar transistors, commonly using +5V supply.

Long Answer Questions (5–7 Marks Style)

Q 7.L1

Explain Boolean algebra. State and explain important Boolean laws with examples.

Boolean algebra is used to analyze and simplify digital logic expressions using two values: 0 and 1.

Important laws:

  1. Identity laws: A + 0 = A, A·1 = A
  2. Null laws: A + 1 = 1, A·0 = 0
  3. Complement laws: A + A̅ = 1, A·A̅ = 0
  4. Idempotent laws: A + A = A, A·A = A
  5. Commutative laws: A + B = B + A, AB = BA
  6. Associative laws: (A + B) + C = A + (B + C)
  7. Distributive law: A(B + C) = AB + AC
  8. Absorption law: A + AB = A

Example: AB + AB̅ = A(B + B̅) = A·1 = A

These laws are used to reduce hardware cost by simplifying logic expressions.

Q 7.L2

Explain all basic logic gates with truth tables.

Basic logic gates: AND, OR, NOT, NAND, NOR, XOR, XNOR.

  • AND: Output 1 only when all inputs are 1
  • OR: Output 1 when at least one input is 1
  • NOT: Output is complement of input
  • NAND: Complement of AND
  • NOR: Complement of OR
  • XOR: Output 1 when inputs are different
  • XNOR: Output 1 when inputs are same

Truth tables should be written for each gate in exam. Refer to the gate section above for complete truth tables.

Important point: NAND and NOR are universal gates.

Q 7.L3

What are universal gates? Show how basic gates can be realized using NAND gates.

Universal gates are gates that can realize all other logic gates. NAND and NOR are universal gates.

Using NAND gate:

  1. NOT: Y = (A·A)̅ = A̅
  2. AND: First NAND gives (AB)̅, second NAND used as inverter → AB
  3. OR: By De-Morgan: A + B = (A̅·B̅)̅, where A̅ and B̅ are produced using NAND-as-NOT

Therefore, using only NAND gates, all other gates can be obtained.

Q 7.L4

State and prove De-Morgan’s theorems.

De-Morgan’s first theorem:

(AB)̅ = A̅ + B̅

De-Morgan’s second theorem:

(A + B)̅ = A̅B̅

Proof: Both theorems are proved using truth tables.

For (AB)̅ and A̅ + B̅, the output columns are identical for all 4 input combinations.

For (A+B)̅ and A̅B̅, the output columns are also identical for all input combinations.

Hence both theorems are proved.

Use: These theorems are very useful in converting logic circuits into NAND/NOR implementation.

Q 7.L5

Explain minterm, maxterm, SOP and POS with suitable examples.

Minterm: Standard product term with all variables once. Example for A,B: A̅B is m1.

Maxterm: Standard sum term with all variables once. Example for A,B: A + B̅ is M1.

SOP: Sum of Products — OR of product terms. Example: F = A̅B + AB̅ + AB

POS: Product of Sums — AND of sum terms. Example: F = (A + B)(A̅ + B)(A + B̅)

Important relation:

  • SOP is based on rows where output = 1
  • POS is based on rows where output = 0

Notation: F = Σm(...) for SOP and F = ΠM(...) for POS.

Q 7.L6

Explain K-Map simplification method with rules.

K-Map is a graphical method for simplifying Boolean expressions.

Rules:

  1. Place 1s in cells corresponding to minterms
  2. Group adjacent 1s in groups of 1, 2, 4, 8... only
  3. Make the largest possible groups
  4. Each group gives one simplified term
  5. Corner cells are adjacent
  6. Cells can be reused if necessary

Advantage: Gives minimal expression quickly and reduces number of gates.

K-Maps are most suitable for 2, 3 and 4 variable problems.

Q 7.L7

Explain the basic concept of flip-flops as storage elements.

A flip-flop is a bistable sequential circuit used to store one bit of data.

Why called bistable? Because it has two stable states: 0 and 1.

Basic types:

  • SR flip-flop: Set and Reset operations
  • JK flip-flop: Improved SR flip-flop, no invalid state
  • D flip-flop: Stores data directly, Q(next) = D
  • T flip-flop: Toggles when T = 1

Applications: Counters, registers, memory units, sequence generators.

Q 7.L8

Compare ripple counter and synchronous counter. Also explain TTL in brief.

Ripple Counter:

  • Asynchronous counter
  • Clock goes only to first flip-flop
  • Slower due to ripple delay
  • Simple design

Synchronous Counter:

  • All flip-flops get clock together
  • Faster operation
  • Used in high-speed systems
  • More complex design

TTL: TTL stands for Transistor-Transistor Logic. It is a BJT-based logic family operating usually at +5V. Common TTL ICs are from 74xx series such as 7400, 7402, 7404 etc.
