What is an Operating System?

Definition

Operating System (OS) is system software that manages computer hardware, software resources, and provides common services for computer programs. It acts as an intermediary between users and the hardware.

Computer System Review

A computer system consists of hardware (CPU, memory, I/O), operating system, application programs, and users. The OS controls and coordinates the hardware among applications and users.

Hardware ↔ OS ↔ Applications ↔ Users

Kernel – the core

The kernel is the central component of an OS, always resident in memory. It handles process management, memory management, device drivers, system calls.

Brief History & Background

1940s-50s: No OS

Programmers physically connected wires, plugboards. Later, resident monitor loaded programs sequentially.

1960s: Batch systems

Jobs grouped into batches, reduced setup time. IBM OS/360, etc.

1970s: Timesharing

Multiple interactive users. UNIX, Multics. CTSS was early.

1980s–now: PC, GUI, mobile

MS‑DOS, Mac OS, Windows, Linux, Android, iOS.

Background drivers

Hardware evolution (transistors → ICs → microprocessors) enabled more sophisticated OS. Need for efficiency, resource sharing, user convenience shaped OS history.

Think: Which decade introduced the first GUI-based OS for personal computers?

Answer: Xerox Alto (1973) was the first with GUI, but it wasn't for personal computers. The first GUI-based OS for personal computers was:

• Windows 1.0 (1985) - Microsoft
So the 1980s is the correct decade for personal computer GUIs.

Types of Operating Systems

Basic OS Types
  • Batch OS – Jobs with similar needs grouped, minimal user interaction.
  • Multi-programmed Batch – Several jobs in memory, CPU switched to keep it busy.
  • Timesharing (multitasking) – CPU switches rapidly among jobs, giving interactive users fast response.

Batch OS

Operator batches jobs (e.g., Fortran + data). No interaction. Slow turnaround. Example: early IBM mainframes.

Multi‑programmed batch

Multiple jobs in memory. When one waits for I/O, OS switches to another. Increases CPU utilization.

Timesharing

Extension of multiprogramming. Each user has a terminal. OS schedules CPU in small time slices. Unix, Linux, Windows.

Architecture / OS structures (preview)

OS can be designed as monolithic (Linux), layered, microkernel (QNX, macOS partially), hybrid (Windows NT), or exokernel (research).

Computer System Structures

I/O structure: after I/O starts, control returns to user program only after completion (synchronous) or with interrupt (asynchronous). DMA used for high-speed I/O.

Storage structure: memory hierarchy: registers, cache, main memory, solid‑state disk, magnetic disk, optical/tapes.

Exercise: List one advantage of timesharing over simple batch systems.

Advantages of Timesharing over Batch:
• Interactive response – users get immediate feedback
• Multi-user support – many users can share the system simultaneously
• Better resource utilization – CPU switches between active users
• Faster debugging – programmers can test interactively
• Batch systems have no interaction and long turnaround time (hours or days)

Operating System Structures

Common OS structures
  • Simple / Monolithic: whole OS in kernel space (e.g., Linux, older Unix). High performance, but less reliable (any bug crashes kernel).
  • Layered approach: OS divided into layers (bottom: hardware, top: user interface). Each layer uses services of lower layer. (e.g., THE system).
  • Microkernel: minimal kernel – only essential services (IPC, low-level memory). Other services run as user processes (e.g., Mach, QNX).
  • Hybrid (modern): combine monolithic and microkernel for speed + modularity (Windows NT, macOS).

Monolithic

All OS services in kernel. System calls are expensive but fast. Linux, BSD.

Layered

N layer only uses N-1. Debugging easier, but less efficient. Rare today.

Microkernel

Small kernel, services like file system, drivers in user space. More stable, secure. Potential performance overhead.

Example: Windows NT structure

Hybrid: executive runs in kernel mode (contains kernel, HAL, device drivers, object manager, etc.) but also provides environment subsystems (Win32, POSIX) in user mode.

System Calls & API

System calls provide interface to OS services (e.g., read(), fork(), ioctl()). Programmers use API (Win32, POSIX) rather than direct system calls.

user program → API → system call → kernel
OS Structures: boot process

Bootstrap loader (firmware) loads kernel into memory, then kernel initializes data structures, device drivers, and starts first process (init/systemd).

Discussion: Why is a microkernel considered more secure than a monolithic kernel?

Microkernel Security Advantages:
• Minimal privileged code – only essential services run in kernel mode
• Isolation – drivers and file systems run as user processes with limited privileges
• Fault containment – if a service crashes, it doesn't bring down the entire kernel
• Reduced attack surface – smaller code base in kernel means fewer vulnerabilities
• Better access control – communication via IPC with strict checking
