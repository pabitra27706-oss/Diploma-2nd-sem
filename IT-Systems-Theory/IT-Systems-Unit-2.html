What is an Operating System? ⭐🔥

Definition: Operating System (OS)

An Operating System (OS) is a system software that acts as an intermediary (interface) between the computer user and the computer hardware. It manages all hardware resources, provides common services for application programs, and makes the computer convenient and efficient to use.

Real-World Analogy

Think of the Operating System as the manager of a restaurant:

  • The customers (users) don't go into the kitchen directly — they interact with the manager (OS)
  • The kitchen (hardware — CPU, memory, disk) prepares the food
  • The chef & cooks (programs/processes) do the actual cooking
  • The manager (OS) takes orders from customers, assigns work to chefs, manages table allocation (memory), handles billing (file management), and keeps everything running smoothly

Without a manager, the restaurant would be chaotic. Without an OS, the computer can't function!

2.1 Goals of an Operating System

Two Main Goals
  1. Convenience: Make the computer easy and comfortable for the user to use. Users don't need to know hardware details — the OS hides complexity.
  2. Efficiency: Manage hardware resources (CPU, memory, disk, I/O devices) in an efficient and fair manner so that all programs and users get proper service.

2.2 Functions of an Operating System 🔥

1. Process Management

The OS creates, schedules, and terminates processes (running programs). It decides which process gets the CPU and for how long. It handles multitasking — running multiple programs at the same time.

2. Memory Management

The OS manages RAM (main memory). It allocates memory to programs when they start, tracks which parts of memory are in use and which are free, and deallocates memory when programs finish.

3. File Management

The OS organizes data into files and folders (directories). It handles creating, deleting, reading, writing, and searching files. It also controls file access permissions (who can read/write).

4. Storage/Disk Management

The OS manages secondary storage (HDD, SSD). It handles disk space allocation, disk scheduling (which data to read/write first), and free space management.

5. I/O Device Management

The OS communicates with input/output devices (keyboard, mouse, printer, monitor) through device drivers. It handles data transfer between devices and memory.

6. Security & Protection

The OS protects the system from unauthorized access using passwords, user accounts, and permissions. It prevents one program from interfering with another.

7. Networking

The OS manages network connections — allowing the computer to communicate with other computers, share files, and access the internet.

8. Error Detection & Handling

The OS detects errors in CPU, memory, I/O devices, and user programs. It takes appropriate action — like displaying an error message or terminating the faulty program.

9. User Interface

The OS provides a way for users to interact with the computer. CLI (Command Line Interface): text commands (e.g., Linux terminal). GUI (Graphical User Interface): windows, icons, menus (e.g., Windows desktop).

Exam Must-Know

When asked "What are the functions of an OS?" — write at least 6 functions with brief explanation. The most commonly asked are: Process Management, Memory Management, File Management, I/O Management, Security, and User Interface.

2.4 Different Views of an Operating System

How to Look at an OS
  • As a Resource Manager/Allocator: The OS manages all hardware resources (CPU, memory, disk, I/O devices) and allocates them to various programs/users in a fair and efficient manner. When multiple programs compete for the same resource, the OS decides who gets it.
  • As a Control Program: The OS controls the execution of user programs and operation of I/O devices. It prevents errors and improper use of the computer.
  • As an Extended Machine / Virtual Machine: The OS hides the ugly details of hardware and provides a simple, clean interface. For example, instead of knowing how disk sectors work, you just "save a file" — the OS handles the rest.
Key Points to Remember
  • OS is the first software loaded when a computer starts (after BIOS)
  • OS runs in kernel mode (privileged mode) with full access to hardware
  • User programs run in user mode with restricted access
  • The process of loading the OS into memory is called booting
  • The core part of the OS is called the kernel

Brief History of Operating Systems

Evolution of Operating Systems

Operating Systems didn't appear overnight. They evolved over several decades, with each generation solving the problems of the previous one. Understanding this history helps you understand WHY different types of OS were created.

🔧 Generation 0 — No OS (1940s)

Computers: ENIAC, EDVAC (huge room-sized machines using vacuum tubes)

Programming: Done by directly wiring circuits or using machine language (binary). No operating system existed.

Problem: Extremely difficult to use. Only scientists and engineers could operate them. Programs had to be loaded manually using punch cards or switches.

Key fact: Each program had complete control over the entire machine. One user at a time.

📋 Generation 1 — Batch Systems (1950s)

Computers: IBM 701, IBM 7094 (transistor-based)

Key Innovation: Introduction of Batch Processing. An operator would collect similar jobs (programs), group them into a "batch," and run them one after another without user interaction.

How it worked:

  1. Users submit jobs (programs + data) on punch cards to an operator
  2. Operator groups similar jobs into a batch
  3. Batch is loaded into the computer
  4. A simple program called Resident Monitor controlled execution
  5. Jobs executed one by one — output collected on tape/paper

Problem: CPU was idle during I/O operations (reading cards, printing). Very wasteful!

⚡ Generation 2 — Multiprogrammed Batch Systems (1960s)

Computers: IBM System/360 (Integrated Circuits)

Key Innovation: Multiprogramming — multiple programs loaded into memory simultaneously. When one program waits for I/O, the CPU switches to another program. CPU is never idle!

Also introduced: Spooling (Simultaneous Peripheral Operations On-Line) — using disk as buffer between slow I/O devices and fast CPU.

Problem: No user interaction possible during execution. Users had to wait for batch results.

👥 Generation 3 — Timesharing / Multitasking Systems (1970s)

Computers: DEC PDP, early Unix machines

Key Innovation: Timesharing — the CPU time is divided into small slices (time quantum) and shared among multiple users. Each user gets a terminal and feels like they have the entire computer to themselves.

Important OS: UNIX was developed at AT&T Bell Labs by Ken Thompson and Dennis Ritchie (1969-1971). It became the foundation for many modern operating systems.

Also: Interactive computing became possible — users could type commands and get immediate responses.

💻 Generation 4 — Personal Computer OS (1980s–Present)

Computers: IBM PC, Apple Macintosh, modern PCs

Key Innovation: Development of GUI-based operating systems for personal computers.

Key milestones:

  • 1981: MS-DOS released by Microsoft (CLI-based)
  • 1984: Apple Macintosh — first popular GUI-based computer
  • 1985: Windows 1.0 by Microsoft (GUI on top of DOS)
  • 1991: Linux kernel created by Linus Torvalds (open source)
  • 1995: Windows 95 — huge commercial success
  • 2001: Windows XP, Mac OS X
  • 2007: iPhone & iOS — birth of modern mobile OS
  • 2008: Android OS released (based on Linux kernel)
  • 2015: Windows 10
  • 2021: Windows 11

Computer System Review

Why Review Computer Systems?

Before understanding how an OS works, we need to understand the hardware it manages. This section reviews the basic components and organization of a modern computer system from the OS perspective.

4.1 Basic Computer System Organization

Computer System Components

A modern computer system consists of:

  • One or more CPUs (Processors)
  • Main Memory (RAM)
  • I/O Devices (disk, keyboard, display, network, etc.)
  • System Bus — connects all components and allows communication
  • Device Controllers — manage specific I/O devices

4.2 Components Explained (From OS Perspective)

1. CPU (Central Processing Unit)

The brain of the computer. From the OS point of view:

  • Executes instructions from programs one by one
  • Has two modes of operation:
    • Kernel Mode (Privileged Mode): OS runs here. Has full access to all hardware and instructions.
    • User Mode: User programs run here. Restricted access — cannot directly access hardware.
  • Mode Bit: A hardware bit that distinguishes between kernel mode (0) and user mode (1)
  • The CPU switches between these modes when a program makes a system call (requesting OS service)
2. Main Memory (RAM)

From the OS perspective:

  • Only storage that the CPU can directly access
  • Programs must be loaded into RAM before the CPU can execute them
  • The OS is responsible for:
    • Tracking which parts of memory are being used and by which process
    • Allocating and deallocating memory as needed
    • Deciding which processes to load when memory is limited
  • Volatile — contents lost when power off
3. Secondary Storage

Since RAM is volatile and limited, the computer needs non-volatile, large-capacity storage:

  • Hard Disk Drive (HDD): Stores OS, programs, data permanently
  • Solid State Drive (SSD): Faster alternative to HDD
  • The OS manages the file system on secondary storage
  • Data is organized in files and directories on the disk
4. I/O Devices & Device Controllers

Each I/O device (keyboard, disk, printer, network card) is connected through a device controller:

  • Device Controller: A hardware component that manages a specific type of device. It has its own local buffer (small memory).
  • Device Driver: A software module (part of OS) that communicates with the device controller. Each device needs its own driver.
  • How I/O works:
    1. CPU tells device controller what to do (via device driver)
    2. Device controller performs the operation and stores data in its local buffer
    3. Device controller informs CPU via an interrupt when done
    4. CPU/OS transfers data between controller buffer and main memory
5. System Bus

The system bus is the communication pathway that connects CPU, memory, and I/O devices. It consists of three types:

  • Data Bus: Carries actual data being transferred
  • Address Bus: Carries the memory address where data should go or come from
  • Control Bus: Carries control signals (read, write, interrupt signals)

4.3 Interrupts ⭐

Definition: Interrupt

An interrupt is a signal sent to the CPU by hardware or software indicating that an event has occurred that needs immediate attention. When an interrupt occurs, the CPU stops its current task, saves its state, handles the interrupt, and then resumes the original task.

Analogy

You are studying (CPU executing a program). Your phone rings (interrupt). You pause studying, pick up the phone (handle interrupt), finish the call, then resume studying from where you left off. That's exactly how interrupts work!

Types of Interrupts:

TypeAlso CalledCaused ByExample
Hardware InterruptExternal InterruptI/O devices, hardware signalsKeyboard key press, disk read complete, timer expires
Software InterruptTrap / ExceptionProgram instructions, errorsSystem call, division by zero, invalid memory access
How Interrupt Handling Works
  1. An interrupt signal arrives at the CPU
  2. CPU finishes current instruction
  3. CPU saves its current state (register values, program counter) onto the stack
  4. CPU identifies the interrupt source using the Interrupt Vector Table (IVT)
  5. CPU jumps to the appropriate Interrupt Service Routine (ISR)
  6. ISR handles the interrupt (e.g., reads data from keyboard buffer)
  7. CPU restores saved state and resumes the interrupted program

4.4 DMA (Direct Memory Access)

Definition: DMA

DMA (Direct Memory Access) is a feature that allows I/O devices to transfer data directly to/from main memory without involving the CPU for every byte of data. The CPU only initiates the transfer and is notified when it's complete. This frees up the CPU to do other work during data transfer.

Why DMA is Important

Without DMA: CPU has to handle every single byte of data transfer between I/O device and memory → CPU is busy and can't do other work.

With DMA: A special DMA controller handles the data transfer. CPU just tells DMA controller: "Transfer this block of data from disk to memory location X." CPU goes back to executing other programs. DMA controller sends an interrupt when transfer is complete.

DMA is essential for high-speed devices like hard disks, SSDs, and network cards.

4.5 Booting Process

Definition: Booting

Booting is the process of starting a computer and loading the operating system from secondary storage (disk) into main memory (RAM). The term comes from "pulling yourself up by your bootstraps."

Steps of Booting
  1. Power ON: Electricity flows to all components
  2. BIOS/UEFI starts: The first program that runs — stored in ROM. It performs POST (Power-On Self Test) — checks if CPU, RAM, keyboard, display are working.
  3. Boot Loader: BIOS looks for a bootable device (HDD, SSD, USB) and loads the boot loader (small program on disk)
  4. OS Kernel loads: Boot loader loads the OS kernel into RAM
  5. OS initializes: Kernel starts system services, drivers, and finally presents the login screen / desktop to the user
TypeDescription
Cold BootStarting a computer from completely powered-off state (pressing power button)
Warm BootRestarting a computer that is already running (Ctrl+Alt+Del or Restart)
Remember for Exam
  • BIOS = Basic Input/Output System (stored in ROM)
  • UEFI = Unified Extensible Firmware Interface (modern replacement for BIOS)
  • POST = Power-On Self Test
  • Boot Loader examples: GRUB (Linux), Windows Boot Manager
  • Kernel = Core part of OS loaded first into memory

5.1 Batch Operating System 🔥

Definition: Batch Operating System

A Batch Operating System is one where similar jobs (programs) are collected together into a batch and submitted to the computer for processing one after another without user interaction. The user does not interact with the computer directly — jobs are submitted and results are collected later.

Analogy

Think of a laundry service. You drop off all your dirty clothes (jobs) in the morning. The laundry collects clothes from many customers, groups similar items (batch), washes them one batch at a time, and you collect your clean clothes (results) in the evening. You don't stand and watch each shirt being washed — there's no interaction!

How Batch OS Works
  1. Users prepare their jobs on punch cards or submit job files
  2. An operator collects all jobs and groups similar ones into a batch
  3. The batch is given to the computer
  4. A special program called Resident Monitor (a simple OS) automatically loads and runs jobs one by one
  5. Each job runs to completion before the next job starts
  6. Output is collected on printer or tape — user retrieves results later

Fig. 5.2: Batch OS Working — User submits jobs → Operator batches → Computer processes sequentially → Output collected

🔍 Click to view on Google Images

Advantages of Batch OS:

  • Simple to implement and understand
  • Reduces setup time by grouping similar jobs
  • Can process large volumes of repetitive work efficiently
  • Operator can prioritize batches

Disadvantages of Batch OS:

  • No user interaction — user cannot modify job during execution
  • CPU idle during I/O — when a job does input/output, CPU waits and wastes time
  • Long turnaround time — user has to wait hours or even days for results
  • Debugging is difficult — errors found only after batch completes
  • One job failure can delay entire batch
Real-World Applications of Batch OS
  • Bank statement generation (processing millions of accounts overnight)
  • Payroll processing in large companies
  • Bill printing for electricity/telephone companies
  • Scientific data processing (weather forecasting)
  • Report generation from large databases

5.2 Multiprogrammed Operating System 🔥

Definition: Multiprogrammed OS

A Multiprogrammed Operating System loads multiple programs (jobs) into main memory simultaneously. When one program is waiting for an I/O operation (like reading from disk), the CPU switches to another program instead of sitting idle. This maximizes CPU utilization.

Analogy

Imagine a doctor in a clinic. While one patient goes for an X-ray (I/O wait), the doctor doesn't sit idle — they see the next patient. When the X-ray is ready (I/O complete), the first patient comes back. The doctor keeps switching between patients to stay busy all day. That's multiprogramming!

How Multiprogrammed OS Works
  1. Multiple jobs are loaded into different parts of main memory
  2. CPU starts executing Job 1
  3. When Job 1 requests I/O (e.g., read from disk), it has to wait
  4. Instead of waiting, OS switches CPU to Job 2
  5. When Job 2 also requests I/O, CPU switches to Job 3
  6. When Job 1's I/O is complete, it becomes ready again
  7. CPU keeps switching between ready jobs — never idle!

Fig. 5.3: Multiprogramming — Multiple jobs in memory, CPU switches when one job waits for I/O

🔍 Click to view on Google Images

Advantages of Multiprogrammed OS:

  • High CPU utilization — CPU is almost never idle
  • Better throughput (more jobs completed per unit time)
  • Shorter overall job completion time compared to batch
  • Efficient use of system resources

Disadvantages of Multiprogrammed OS:

  • Requires more memory (to hold multiple jobs)
  • Complex memory management needed (memory protection, allocation)
  • CPU scheduling algorithms needed to decide which job runs next
  • Still no user interaction during execution
  • One job can monopolize the CPU if it doesn't do I/O
Key Difference from Batch

In Batch OS: Only ONE job in memory at a time. CPU idle during I/O.
In Multiprogrammed OS: MULTIPLE jobs in memory. CPU switches to another job during I/O. CPU utilization is much higher!

5.3 Timesharing (Multitasking) Operating System 🔥

Definition: Timesharing OS

A Timesharing Operating System (also called Multitasking OS) allows multiple users to use the computer simultaneously by dividing CPU time into small intervals called time slices (time quantum). The CPU rapidly switches between users/tasks, giving each one a small time slot. Each user feels like they have the entire computer to themselves.

Analogy

Think of a teacher in a classroom helping 30 students. The teacher goes to Student 1, helps for 1 minute, then moves to Student 2 for 1 minute, then Student 3, and so on. The teacher cycles through all students repeatedly. Each student feels like they're getting individual attention because the teacher moves so fast. That's timesharing!

How Timesharing OS Works
  1. Multiple users connect to the same computer via terminals (screens + keyboards)
  2. Each user's task (process) is loaded into memory
  3. The CPU allocates a small time quantum (typically 10–100 milliseconds) to each user's task
  4. After the time quantum expires, the OS preempts (forcefully takes away) the CPU from the current task
  5. CPU switches to the next user's task
  6. This round-robin switching happens so fast that each user feels like they have the full computer
  7. Users can interact with their programs in real-time (type commands, see output immediately)

Fig. 5.4: Timesharing OS — Multiple users connected via terminals, CPU time divided into slices

🔍 Click to view on Google Images

Key Concepts in Timesharing:

Important Terms
  • Time Quantum (Time Slice): The small fixed amount of CPU time given to each task before switching. Typically 10–100 milliseconds.
  • Context Switching: The process of saving the state of the current task and loading the state of the next task. Happens every time CPU switches between tasks.
  • Preemption: Forcefully taking the CPU away from a task when its time quantum expires, even if the task isn't finished yet.
  • Response Time: The time between a user typing a command and seeing the response. In timesharing, response time should be very short (interactive).
  • Round Robin Scheduling: The scheduling algorithm used in timesharing — tasks are served in circular order, each getting one time quantum per turn.

Advantages of Timesharing OS:

  • User interaction possible — users can type commands and get immediate responses
  • Multiple users can use the same computer simultaneously
  • Quick response time — feels interactive
  • Fair CPU allocation — each user gets equal time
  • Reduces idle CPU time
  • Each user feels like they have the whole computer

Disadvantages of Timesharing OS:

  • Complex OS design — needs CPU scheduling, memory protection, context switching
  • Context switching overhead — switching between tasks takes time (waste)
  • Security concerns — multiple users sharing system → risk of data access by unauthorized user
  • Reliability issues — if main system fails, ALL users are affected
  • Performance degrades with too many users
  • Requires significant memory to hold all user processes
Examples of Timesharing OS
  • UNIX — the first widely-used timesharing OS
  • Linux — multi-user, timesharing
  • Multics — predecessor of UNIX, designed for timesharing
  • Windows (modern versions) — supports multitasking
  • macOS — supports multiple users and tasks

5.4 Real-Time Operating System (RTOS)

Definition: Real-Time OS

A Real-Time Operating System (RTOS) is an OS that guarantees processing of events or data within a strict, defined time limit (deadline). If the system doesn't respond within the deadline, it is considered a failure. Used in systems where timing is critical.

Types of Real-Time OS:

TypeDeadlineFailure ConsequenceExamples
Hard Real-TimeAbsolutely strict — cannot missCatastrophic (life-threatening)Airbag systems, pacemakers, missile guidance, ABS brakes, nuclear reactor control
Soft Real-TimePreferred but can occasionally missDegraded quality (not fatal)Video streaming, online gaming, VoIP calls, multimedia playback

Characteristics of RTOS:

  • Deterministic: Response time is predictable and guaranteed
  • Low latency: Very fast response to events
  • Priority-based scheduling: Critical tasks get CPU first
  • Minimal interrupt latency: Interrupts handled almost immediately
  • Small and efficient: Lightweight OS, no unnecessary features
RTOS Examples
  • VxWorks — used in Mars rovers, Boeing 787
  • FreeRTOS — open-source RTOS for microcontrollers (IoT)
  • QNX — used in car infotainment systems, BlackBerry
  • RTLinux — real-time version of Linux

5.5 Distributed Operating System

Definition: Distributed OS

A Distributed Operating System manages a group of independent computers (nodes) connected through a network and makes them appear to the user as a single coherent system. Resources (CPU, memory, storage) of multiple machines are shared and coordinated by the OS.

Characteristics:

  • Multiple computers work together as one system
  • Resource sharing across network
  • Transparency: User doesn't know which computer is processing their task
  • Fault tolerance: If one computer fails, others take over
  • Scalability: Can add more computers to increase performance

Examples:

  • LOCUS, Amoeba (research distributed OS)
  • Google's infrastructure (uses distributed computing)
  • Cloud computing platforms (AWS, Azure, Google Cloud)

Advantages:

  • High reliability and fault tolerance
  • Resource sharing reduces cost
  • Scalable — add more machines as needed
  • Faster processing through parallel computation

Disadvantages:

  • Complex to design and implement
  • Network dependency — if network fails, system suffers
  • Security challenges across network
  • Difficult to manage and debug

5.6 Network Operating System

Definition: Network OS

A Network Operating System (NOS) runs on a server and provides services to other computers (clients) connected via a network. Unlike a distributed OS, each computer has its own OS and is aware that it's accessing a remote server.

Features:

  • Client-server architecture
  • File sharing, printer sharing over network
  • User account management centralized on server
  • Remote access capabilities

Examples:

  • Windows Server (2016, 2019, 2022)
  • Linux Server (Ubuntu Server, CentOS, Red Hat)
  • Novell NetWare (older)

5.7 Embedded Operating System

Definition: Embedded OS

An Embedded Operating System is a specialized OS designed to run on embedded systems — small, dedicated computers built into devices that perform a specific function. These are not general-purpose computers.

Characteristics:

  • Small in size, optimized for specific hardware
  • Limited resources (low memory, low power)
  • Often real-time capable
  • Runs continuously without user intervention
  • Stored in ROM or flash memory

Examples of Devices with Embedded OS:

  • Washing machines, microwave ovens, AC controllers
  • ATM machines, digital cameras
  • Car engine control systems, GPS navigation
  • Smart TVs, set-top boxes
  • Traffic light controllers, elevator controllers

Embedded OS Examples:

  • Embedded Linux
  • Windows IoT (Internet of Things)
  • FreeRTOS
  • Android Things

5.8 Mobile Operating System

Definition: Mobile OS

A Mobile Operating System is an OS specifically designed for smartphones, tablets, and wearable devices. It is optimized for touchscreen input, battery efficiency, mobile connectivity (cellular, Wi-Fi), and app-based usage.

Operating System Architecture 🔥

What is OS Architecture?

OS architecture refers to the internal structure — how the different components of the operating system are organized and how they interact with each other. Different architectures have different trade-offs between performance, simplicity, security, and maintainability.

6.1 Monolithic Architecture (Simple Structure) ⭐

Definition: Monolithic Architecture

In a Monolithic architecture, the entire operating system runs as a single large program in kernel mode. All OS services — process management, memory management, file system, device drivers, networking — are part of one big kernel with no clear separation between modules.

Analogy

Think of a one-room house where the kitchen, bedroom, living room, and bathroom are all in the same room. Everything is together — convenient and fast to move around, but messy and hard to maintain. If the kitchen catches fire, the entire house is at risk!

Characteristics
  • All OS functions run in kernel space (privileged mode)
  • Any function can call any other function directly — fast
  • No clear boundaries between components
  • A bug in any part can crash the entire OS
  • Difficult to modify or extend

Advantages:

  • High performance — direct function calls, no overhead
  • Simple design (conceptually)
  • Fast communication between components

Disadvantages:

  • Hard to maintain — any change requires recompiling entire kernel
  • Less reliable — a bug in driver can crash whole OS
  • Security risk — everything runs in privileged mode
  • Difficult to debug
Examples

Linux kernel (mostly monolithic, but with loadable modules), MS-DOS, early UNIX

6.2 Layered Architecture ⭐

Definition: Layered Architecture

In a Layered architecture, the OS is divided into several layers (levels), where each layer is built on top of the layer below it. Each layer provides services to the layer above and uses services from the layer below. The lowest layer (Layer 0) is the hardware, and the highest layer is the user interface.

Analogy

Think of a multi-story building. The ground floor (hardware) supports the first floor (drivers), which supports the second floor (memory management), and so on. Each floor only interacts with the floor directly above and below it — not with distant floors.

Typical Layers
  • Layer 0: Hardware
  • Layer 1: CPU Scheduling
  • Layer 2: Memory Management
  • Layer 3: Device Drivers (I/O management)
  • Layer 4: File System
  • Layer 5: User Interface / Application Programs

Fig. 6.2: Layered OS Architecture — showing layers from Hardware (bottom) to User Interface (top)

🔍 Click to view on Google Images

Advantages:

  • Modularity: Clear separation between layers — easy to understand
  • Easy debugging: If a problem occurs, you know which layer has the bug
  • Easy modification: Can change one layer without affecting others
  • Easier to design and test — each layer independently

Disadvantages:

  • Slower performance: A request must pass through multiple layers (overhead)
  • Difficult to define layers: Hard to decide which function goes in which layer
  • Less flexible: A layer can only use services of the layer directly below it
Examples

THE OS (by Dijkstra, 1968 — first layered OS), MULTICS, OpenVMS

6.3 Microkernel Architecture ⭐🔥

Definition: Microkernel

In a Microkernel architecture, the kernel is made as small as possible. Only the most essential services (inter-process communication, basic scheduling, basic memory management) run in kernel mode. All other services (file system, device drivers, networking) run as user-space processes called servers.

Analogy

Think of a CEO who delegates everything. The CEO (microkernel) only handles the most critical decisions (communication between departments). All actual work (file management, device handling) is done by department heads (user-space servers). If one department fails, the CEO and other departments continue working.

Fig. 6.3: Microkernel Architecture — small kernel with services running in user space

🔍 Click to view on Google Images

What's IN the Microkernel:

  • Inter-Process Communication (IPC) — message passing
  • Basic CPU scheduling
  • Basic memory management

What's OUTSIDE (in user space):

  • File system server
  • Device driver servers
  • Network server
  • Security server

Advantages:

  • More reliable: If a server crashes, rest of system continues
  • More secure: Less code in privileged mode
  • Easier to extend: Add new services without modifying kernel
  • Portable: Small kernel is easy to port to new hardware

Disadvantages:

  • Slower: Communication between user-space servers requires message passing (overhead)
  • More complex IPC: Need efficient inter-process communication
  • Performance overhead compared to monolithic
Examples

Mach (used as base for macOS), MINIX (educational OS by Andrew Tanenbaum), QNX, L4, GNU Hurd

6.4 Hybrid Kernel Architecture

Definition: Hybrid Kernel

A Hybrid Kernel combines features of both monolithic and microkernel architectures. Some services (like device drivers) run in kernel space for performance, while others run in user space for reliability. It tries to get the best of both worlds.

Characteristics:

  • Core services in kernel space (fast)
  • Some services in user space (reliable)
  • Flexible design — can choose where to put each service
  • Practical compromise between speed and reliability
Examples

Windows NT/10/11 (hybrid kernel), macOS (XNU kernel — Mach microkernel + BSD monolithic), ReactOS

7.3 Key Concepts Explained

Turnaround Time

Turnaround Time = Time from job submission to job completion. It includes waiting time, execution time, and I/O time. Shorter is better.

Response Time

Response Time = Time from user giving a command to getting the first response. In timesharing, this should be very short (< 1 second) for interactive feel.

Throughput

Throughput = Number of jobs completed per unit time. Higher throughput means the system is processing more work. Multiprogramming improves throughput significantly.

CPU Utilization

CPU Utilization = Percentage of time the CPU is busy doing useful work. In batch OS, CPU utilization is low (40–60%). In multiprogrammed OS, it can be 80–95%.

Spooling

Spooling (Simultaneous Peripheral Operations On-Line) = Using the disk as a large buffer to store data between slow I/O devices and the fast CPU. For example, print jobs are "spooled" to disk first, then printed in the background.

Context Switching

Context Switching = The process of saving the complete state (registers, program counter, stack) of the currently running process and loading the saved state of the next process to be run. It happens during CPU switching between processes. Context switching has an overhead — it takes time but produces no useful work.

Common Exam Confusion — Don't Mix These Up!
  • Multiprogramming: Multiple PROGRAMS in memory, ONE CPU (switching on I/O)
  • Multitasking: Multiple TASKS sharing CPU with time slices (interactive)
  • Multiprocessing: Multiple CPUs working together (true parallelism)
  • Multithreading: Multiple threads within ONE process sharing CPU

Remember: Multi-programming = Many programs, one CPU, switch on wait. Multi-tasking = Many tasks, one CPU, switch on timer. Multi-processing = Many CPUs.

Computer System Structures ⭐

What are Computer System Structures?

This topic covers how the different hardware components of a computer are organized and how they interact from the operating system's perspective. The OS must understand these structures to manage the system effectively.

8.1 I/O Structure

I/O Structure Overview

The I/O structure describes how the CPU communicates with I/O devices (keyboard, disk, printer, network card). Since I/O devices are much slower than the CPU, the OS must handle I/O carefully to avoid wasting CPU time.

Two Types of I/O Operations:

1. Synchronous I/O (Blocking I/O)
  • When a process requests I/O, the CPU waits until the I/O operation completes
  • The process is blocked — it cannot do anything until data arrives
  • Simple but wasteful — CPU sits idle during the wait
  • Only one I/O request can be handled at a time
2. Asynchronous I/O (Non-Blocking I/O)
  • When a process requests I/O, the CPU continues doing other work
  • The I/O device performs the operation independently
  • When I/O is complete, the device sends an interrupt to notify the CPU
  • More efficient — CPU is not wasted
  • Used in multiprogrammed and modern OS

8.3 Protection and Security

Protection

Protection is the mechanism of controlling access of processes and users to the resources defined by a computer system. It ensures that one process cannot interfere with another process or with the OS itself.

Security

Security involves defending the computer system against external and internal threats — unauthorized access, viruses, malware, data theft, and denial-of-service attacks.

Protection Mechanisms in OS
  • Dual-Mode Operation: Kernel mode (privileged) vs User mode (restricted). User programs cannot directly access hardware.
  • Memory Protection: Base and limit registers prevent a process from accessing memory outside its allocated space.
  • Timer Protection: A hardware timer prevents any process from monopolizing the CPU. After time expires, OS takes control back.
  • I/O Protection: All I/O instructions are privileged — only the OS can execute them. User programs must request I/O through system calls.
  • File Protection: Access control lists (ACL) and permissions (read, write, execute) control who can access which files.
  • Authentication: Username/password, biometrics, two-factor authentication to verify user identity.

Operating System Structures ⭐🔥

What This Section Covers

This section deals with the internal organization of the OS — what services it provides, how user programs request those services (system calls), and what system programs are available.

9.1 Operating System Services 🔥

OS Services

Operating System Services are the functions that the OS provides to users and programs to make computing more convenient and efficient.

Services Helpful to Users
  1. User Interface (UI):
    • CLI (Command Line Interface): User types text commands. Examples: CMD in Windows, Terminal in Linux.
    • GUI (Graphical User Interface): User interacts using windows, icons, menus, and pointer (mouse). Examples: Windows Desktop, macOS Finder.
    • Touch Interface: Gesture-based input on touchscreens. Examples: Android, iOS.
  2. Program Execution: OS loads a program into memory and runs it. It also handles program termination (normal or abnormal).
  3. I/O Operations: Programs need to read/write from devices. OS provides this service since user programs cannot directly access devices.
  4. File System Manipulation: Create, delete, read, write, search files and directories. Manage permissions.
  5. Communication: Processes need to exchange data. OS provides:
    • Shared Memory: Two processes share a memory region
    • Message Passing: Processes exchange messages through the OS
  6. Error Detection: OS constantly monitors for errors in CPU, memory, I/O devices, and user programs.
Services for System Efficiency
  1. Resource Allocation: When multiple users/processes are running, OS allocates resources (CPU, memory, files, devices) fairly.
  2. Accounting: OS tracks which users/processes use how much resources. Used for billing in commercial systems.
  3. Protection & Security: Controls access to system resources and defends against unauthorized access.

9.2 System Calls ⭐🔥

Definition: System Call

A system call is a programmatic interface through which a user program requests a service from the operating system kernel. It is the mechanism by which a process running in user mode asks the OS (running in kernel mode) to perform a privileged operation on its behalf.

Analogy

Think of a bank customer. You (user program) want to withdraw money (access hardware resource). But you can't go into the vault yourself (user mode restriction). So you fill out a withdrawal slip (system call) and give it to the bank teller (OS kernel). The teller goes to the vault (hardware), gets your money, and gives it to you. The withdrawal slip is the system call!

How a System Call Works
  1. User program needs OS service (e.g., read a file)
  2. Program makes a system call (e.g., read())
  3. CPU switches from user mode to kernel mode (mode bit changes)
  4. OS identifies which service is requested
  5. OS performs the requested operation (reads data from disk)
  6. OS returns the result to the user program
  7. CPU switches back from kernel mode to user mode

9.3 System Programs

Definition: System Programs

System programs (also called system utilities) are programs that come with the OS and provide a convenient environment for program development and execution. They are NOT part of the kernel but use system calls to provide useful services.

Categories of System Programs
  1. File Management: Programs for creating, deleting, copying, renaming, listing, and printing files. Example: File Explorer (Windows), ls, cp, mv (Linux)
  2. Status Information: Programs that display system info — date, time, memory usage, disk space, logged-in users. Example: Task Manager, top, df
  3. File Modification: Text editors for creating/modifying files. Example: Notepad, nano, vim
  4. Programming Language Support: Compilers, assemblers, interpreters, debuggers. Example: GCC (C compiler), Python interpreter
  5. Program Loading & Execution: Loaders and linkers that load programs into memory. Example: Dynamic linker, boot loader
  6. Communication: Programs for inter-process and inter-system communication. Example: email, web browser, FTP, SSH
Key Distinction

System Calls = Low-level interface (programmer uses in code). Like raw ingredients.
System Programs = Higher-level tools built using system calls. Like ready-made dishes.
Most users interact with system programs, not directly with system calls.

Important Definitions

1. Operating System

System software that acts as an intermediary between the user and computer hardware, managing resources and providing services for application programs.

2. Kernel

The core component of an operating system that has complete control over everything in the system. It manages CPU, memory, devices, and runs in privileged (kernel) mode.

3. Batch Operating System

An OS where similar jobs are grouped into batches and processed sequentially without user interaction. Jobs run one after another automatically.

4. Multiprogramming

A technique where multiple programs are loaded into main memory simultaneously. When one program waits for I/O, the CPU switches to another program, maximizing CPU utilization.

5. Timesharing (Multitasking)

A technique where CPU time is divided into small time slices (time quantum) and shared among multiple users/tasks in round-robin fashion, providing interactive computing.

6. Time Quantum (Time Slice)

A small, fixed amount of CPU time (typically 10–100 ms) allocated to each process in a timesharing system before the OS preempts it and switches to the next process.

7. Context Switching

The process of saving the state (registers, program counter, stack) of the currently running process and restoring the state of the next process to be executed on the CPU.

8. Preemption

The act of forcefully taking the CPU away from a currently running process (before it finishes) to allocate it to another process. Used in timesharing and priority-based scheduling.

9. Process

A program in execution. It includes the program code, current activity (program counter, registers), stack, data section, and heap. A process is the basic unit of work managed by the OS.

10. Interrupt

A signal sent to the CPU by hardware or software indicating that an event needs immediate attention. The CPU pauses current work, handles the interrupt via an ISR, then resumes.

11. System Call

A programmatic interface through which a user program requests a service from the OS kernel. It triggers a mode switch from user mode to kernel mode.

12. DMA (Direct Memory Access)

A hardware feature that allows I/O devices to transfer data directly to/from main memory without CPU involvement for each byte, freeing the CPU for other tasks.

13. Booting

The process of starting a computer and loading the operating system from secondary storage into main memory. BIOS/UEFI initiates POST, then the boot loader loads the OS kernel.

14. Spooling

Simultaneous Peripheral Operations On-Line. Using disk as a large buffer to temporarily store data between slow I/O devices and the fast CPU/memory, enabling overlap of I/O and computation.

15. Real-Time Operating System (RTOS)

An OS that guarantees processing of events within strict, defined time deadlines. Hard RTOS: missing deadline is catastrophic. Soft RTOS: occasional misses are tolerable.

16. Distributed Operating System

An OS that manages multiple independent computers connected via a network, making them appear as a single coherent system to users. Provides resource sharing, fault tolerance, and transparency.

17. Monolithic Kernel

An OS architecture where the entire OS runs as a single large program in kernel mode. All services (process, memory, file, device management) are in one kernel. Fast but less reliable.

18. Microkernel

An OS architecture where the kernel is minimal — only basic services (IPC, scheduling) run in kernel mode. Other services (file system, drivers) run as user-space servers. More reliable but slower.

19. Throughput

The number of processes or jobs completed per unit time. Higher throughput indicates better system performance. Multiprogramming significantly improves throughput.

20. Turnaround Time

The total time from submission of a job/process to its completion. It includes waiting time, execution time, and I/O time. Turnaround Time = Completion Time − Submission Time.

21. Response Time

The time from when a user submits a request (types a command) to when the first response is produced. In timesharing systems, response time should be short for interactive feel.

22. Dual-Mode Operation

A hardware mechanism providing two modes of CPU operation — Kernel Mode (privileged, full access) and User Mode (restricted access). Protects OS from faulty or malicious user programs.

Short Answer Questions (2-Mark Style)

Q1. What is an Operating System? State its two main goals.

An Operating System (OS) is system software that acts as an interface between the user and hardware, managing all resources and providing services for applications.

Two main goals:

  1. Convenience: Make the computer easy to use
  2. Efficiency: Manage hardware resources efficiently

Q2. What is the difference between kernel mode and user mode?

Kernel Mode: OS runs here with full access to all hardware and instructions. Can execute any CPU instruction.

User Mode: User programs run here with restricted access. Cannot directly access hardware — must use system calls for OS services.

A mode bit in hardware distinguishes: 0 = Kernel mode, 1 = User mode.

Q3. Define Multiprogramming. What is its main advantage?

Multiprogramming is a technique where multiple programs are loaded into main memory simultaneously. When one program waits for I/O, the CPU switches to another program.

Main advantage: Maximizes CPU utilization — CPU is almost never idle, resulting in higher throughput.

Q4. What is a system call? Give two examples.

A system call is a programmatic interface through which a user program requests a service from the OS kernel. It triggers a mode switch from user mode to kernel mode.

Examples:

  • fork() — creates a new process
  • read() — reads data from a file

Q5. What is an interrupt? Name its two types.

An interrupt is a signal to the CPU indicating that an event needs immediate attention. The CPU pauses its current task, handles the interrupt, then resumes.

Two types:

  1. Hardware Interrupt: Caused by I/O devices (e.g., keyboard press, disk read complete)
  2. Software Interrupt (Trap): Caused by program instructions or errors (e.g., system call, division by zero)

Q6. What is booting? What is the difference between cold boot and warm boot?

Booting is the process of starting a computer and loading the OS into RAM from secondary storage.

Cold Boot: Starting a computer from completely powered-off state (pressing power button).

Warm Boot: Restarting a computer that is already running (Ctrl+Alt+Del or clicking Restart).

Q7. What is context switching?

Context switching is the process of saving the complete state (registers, program counter, stack pointer) of the currently running process and restoring the saved state of another process so it can resume execution on the CPU.

It occurs during CPU switching between processes. It has overhead — takes time but produces no useful computation.

Q8. What is DMA? Why is it important?

DMA (Direct Memory Access) allows I/O devices to transfer data directly to/from main memory without CPU involvement for each byte.

Importance: It frees the CPU from handling every byte of data transfer, allowing the CPU to do other useful work during I/O operations. Essential for high-speed devices like disks and network cards.

Q9. Name any four functions of an Operating System.

  1. Process Management: Creates, schedules, and terminates processes
  2. Memory Management: Allocates and deallocates RAM to programs
  3. File Management: Creates, deletes, reads, writes files and directories
  4. I/O Device Management: Communicates with hardware devices through drivers

Q10. What is spooling? Give an example.

Spooling (Simultaneous Peripheral Operations On-Line) is a technique where disk is used as a buffer between slow I/O devices and the fast CPU.

Example: Print spooling — when you print multiple documents, they are stored (spooled) on the disk in a print queue. The printer prints them one by one from the queue. The CPU doesn't wait for the printer to finish each page.

Long Answer Questions (5–7 Mark Style)

Q1. What is an Operating System? Explain its main functions in detail. [7 marks]

An Operating System (OS) is system software that acts as an interface between users and hardware. It manages all resources and provides a platform for applications.

Main Functions:

1. Process Management: Creates, schedules, terminates processes. Handles multitasking — running multiple programs simultaneously. Uses scheduling algorithms to decide which process gets CPU.

2. Memory Management: Manages RAM allocation. Tracks which memory is used/free. Loads programs into memory and deallocates when finished.

3. File Management: Organizes data into files and directories. Handles creating, deleting, reading, writing files. Controls access permissions.

4. I/O Device Management: Manages communication with hardware devices through device drivers. Handles data transfer between devices and memory.

5. Security & Protection: Controls access using passwords, user accounts, file permissions. Prevents unauthorized access and program interference.

6. User Interface: Provides CLI (Command Line) and GUI (Graphical) for user interaction.

7. Error Detection: Monitors for errors in CPU, memory, I/O devices and takes appropriate corrective action.

Q3. Explain Monolithic, Layered, and Microkernel OS architectures with their advantages and disadvantages. [7 marks]

1. Monolithic Architecture: Entire OS runs as one large program in kernel mode. All services (process, memory, file, device management) are in a single kernel.

  • Advantages: Fast performance (direct function calls), simple concept
  • Disadvantages: Hard to maintain, one bug crashes entire OS, less secure
  • Example: Linux, MS-DOS

2. Layered Architecture: OS divided into several layers. Each layer provides services to the layer above and uses services from the layer below. Layer 0 = Hardware, top layer = User interface.

  • Advantages: Modular, easy debugging, easy modification
  • Disadvantages: Slower (requests pass through many layers), hard to define layers
  • Example: THE OS, MULTICS

3. Microkernel Architecture: Kernel is minimal — only basic services (IPC, scheduling) in kernel. Other services (file system, drivers) run as user-space servers.

  • Advantages: More reliable (server crash isolated), more secure, easier to extend
  • Disadvantages: Slower (message passing overhead), complex IPC design
  • Example: Mach, MINIX, QNX

Q4. Write a short note on Real-Time Operating System (RTOS). Differentiate between Hard and Soft RTOS. [5 marks]

A Real-Time Operating System (RTOS) guarantees processing of events within strict, defined time deadlines. The correctness of the system depends not only on the logical result but also on the time it is produced.

Characteristics: Deterministic response time, low latency, priority-based scheduling, minimal interrupt latency.

Applications: Airbag systems, pacemakers, missile guidance, industrial automation.

Q5. Explain the concept of system calls. What are the different categories of system calls? [5 marks]

A system call is a programmatic interface through which user programs request services from the OS kernel. When a system call is made, the CPU switches from user mode to kernel mode, the OS performs the requested service, and control returns to user mode.

Categories of System Calls:

  1. Process Control: Create/terminate processes — fork(), exec(), exit(), wait()
  2. File Management: Create/open/read/write/close files — open(), read(), write(), close()
  3. Device Management: Request/release devices — ioctl(), read(), write()
  4. Information Maintenance: Get/set system info — getpid(), time()
  5. Communication: Message passing, shared memory — pipe(), socket(), shmget()
  6. Protection: Set file permissions — chmod(), chown()

Q6. Describe the brief history of operating systems from 1940s to present. [5 marks]

1940s (No OS): First electronic computers (ENIAC). No OS existed. Programs loaded manually using machine language and wires.

1950s (Batch Systems): Jobs collected into batches, processed sequentially by a Resident Monitor. IBM mainframes. CPU idle during I/O.

1960s (Multiprogramming): Multiple jobs loaded into memory. CPU switches during I/O waits. Spooling introduced. IBM System/360.

1970s (Timesharing): CPU time shared among users. UNIX developed at Bell Labs (1969-71) by Ken Thompson & Dennis Ritchie. Interactive computing.

1980s-Present (Personal/Mobile OS): MS-DOS (1981), Windows (1985), Linux (1991 by Linus Torvalds), macOS, Android (2008), iOS (2007). GUI-based, multi-user, networked systems.

Key milestones: UNIX (1969), MS-DOS (1981), Linux (1991), Windows 95 (1995), Android (2008).

Q7. Explain the different types of operating systems with examples. [7 marks]

Operating systems are classified as:

1. Batch OS: Jobs grouped and processed sequentially without user interaction. Example: Early IBM systems.

2. Multiprogrammed OS: Multiple jobs in memory, CPU switches on I/O wait. Maximizes CPU utilization. Example: IBM OS/360.

3. Timesharing OS: CPU time divided into slices among multiple users. Interactive. Example: UNIX, Linux.

4. Real-Time OS: Strict time deadlines for task completion. Example: VxWorks (airbags), FreeRTOS.

5. Distributed OS: Multiple computers appear as one system. Example: Cloud computing platforms.

6. Network OS: Server provides services to client computers over network. Example: Windows Server.

7. Embedded OS: Specialized OS for dedicated devices. Example: Washing machine controller, ATM.

8. Mobile OS: Designed for smartphones and tablets. Example: Android, iOS.

Q8. Explain the protection mechanisms used in an operating system. [5 marks]

Protection mechanisms ensure that one process cannot interfere with another or with the OS itself.

1. Dual-Mode Operation: CPU operates in Kernel mode (full access) and User mode (restricted). User programs cannot directly access hardware.

2. Memory Protection: Base and limit registers ensure each process can only access its allocated memory region. Prevents one process from reading/writing another's data.

3. Timer Protection: Hardware timer prevents any process from monopolizing the CPU. When timer expires, OS takes control.

4. I/O Protection: All I/O instructions are privileged — only the OS can execute them. User programs must use system calls.

5. File Protection: Access control through permissions (read, write, execute) and user authentication (password, biometrics).
