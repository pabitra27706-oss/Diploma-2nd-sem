Basic Internet Skills & Digital India

Understanding a browser

Number Systems & Interconversion

Base systems

Decimal (0-9) · Binary (0,1) · Octal (0-7) · Hexadecimal (0-9,A-F).

Conversion example
(1011.01)₂ = 1·2³ + 0·2² + 1·2¹ + 1·2⁰ + 0·2⁻¹ + 1·2⁻² = 8+2+1+0.25 = 11.25₁₀
(2F)₁₆ = 2·16 + 15 = 32+15 = 47₁₀

Binary → Octal

Group 3 bits from radix point: 10 101.01 → 010 101 . 010 = (25.2)₈

Binary → Hex

Group 4 bits: 1010 1101 . 1010 → (AD.A)₁₆

Signed binary & complements

1's complement: invert bits. 2's complement: invert + add 1. Used in ALU for subtraction.

Example: 2's complement of 13 (8 bits)

13₁₀ = 00001101₂ → 1's: 11110010 → +1 = 11110011 (-13).

Write binary of positive number (8 bits)
13 = 00001101
Invert each bit → 1's complement
11110010
Add 1 → 2's complement
11110011

Practice: Convert (5A.8)₁₆ to binary and octal. Then find 2's complement of 35 (use 8 bits).

Solution:
1. 5A.8₁₆ = 01011010.1000₂
2. Group octal: 01 011 010 . 100 → (132.4)₈
3. 35₁₀ = 00100011₂
4. 1's complement: 11011100
5. 2's complement: 11011101 (-35)

Digital Codes & Code Conversion

BCD (8421) & 2421 codes

BCD represents each decimal digit with 4 bits: 8-4-2-1. e.g., 39 → 0011 1001. 2421 is self‑complementing: 2-4-2-1 weights.

Gray code & Cyclic

Gray: consecutive numbers differ by one bit. Useful in encoders. Cyclic code = Gray is a cyclic code. Binary to Gray: MSB same, then XOR consecutive bits.

Gray conversion example

Binary 1011 → Gray: 1 (MSB) → 1⊕0=1 → 0⊕1=1 → 1⊕1=0 → Gray = 1110.

Alphanumeric codes

ASCII (7/8 bit): standard for English. 'A' = 65 (0x41). EBCDIC (IBM mainframe). Unicode (UTF‑8) dominant now.

Exercise: Encode decimal 48 in BCD, Excess-3, and ASCII (as character '0').

Solution:
• BCD: 48 → 0100 1000
• Excess-3: (4+3=7→0111, 8+3=11→1011) → 0111 1011
• ASCII '0' = 48₁₀ = 0x30 = 00110000

Boolean Algebra & Logic Gates

Basic gates

AND, OR, NOT, NAND, NOR, XOR, XNOR. Truth tables essential.

Universal gates

NAND and NOR can implement any Boolean function.

Laws of Boolean algebra

Identity, null, idempotent, complement, involution, commutative, associative, distributive, absorption, De Morgan.

Common mistake

De Morgan: (A·B)' = A' + B' and (A+B)' = A' · B' . Do not forget to complement each variable.

De Morgan simplification
((A·B)' + C)' = (A·B) · C' (by De Morgan & involution)
(X + C)' = X' · C' where X = (A·B)'
((A·B)')' = A·B
result = A·B·C'

Exercise: Simplify Y = (A'·B)' + (A + B')' using Boolean laws.

Step-by-step simplification:
1. (A'·B)' = A + B' (De Morgan)
2. (A + B')' = A'·B (De Morgan)
3. Y = (A + B') + (A'·B)
4. = A + B' + A'·B
5. = A + (B' + A'·B)
6. = A + (B' + A') (by absorption: B' + A'·B = B' + A')
7. = A + A' + B'
8. = 1 + B'
9. = 1 (always true)

Computer Hardware Components

CPU

Brain of computer: CU, ALU, registers. Modern: multi-core, cache (L1,L2,L3).

Memory types

RAM (DRAM, SRAM), ROM (PROM, EPROM, EEPROM), cache, virtual memory.

Display

LCD, LED, OLED, resolution (1080p, 4K), refresh rate, panel types (IPS, TN, VA).

Keyboard & Mouse

Mechanical/membrane, optical/laser mouse, ergonomic, programmable keys.
Storage: HDD vs SSD

HDD magnetic spinning platters, high capacity, cheaper. SSD flash memory, faster, durable, no moving parts. NVMe faster than SATA.

Peripheral devices

Printer (laser/inkjet), scanner, speaker, webcam, external drives, UPS. Interfaces: USB, HDMI, Bluetooth, Thunderbolt.

Types of software

System software (OS, drivers, utilities) · Application software (word, browser, games) · Programming tools (compilers, IDEs).

Identify your hardware

Use Task Manager (Windows) or System Information to see CPU, RAM, disk type. For Linux: lscpu, lsblk, lspci.

Research: Find out the difference between UEFI and BIOS. Which memory is faster: DDR4 or DDR5?

Answer:
• UEFI (Unified Extensible Firmware Interface) is the modern replacement for BIOS. It supports secure boot, larger hard drives, faster booting, and a graphical interface.
• DDR5 is faster and more power-efficient than DDR4. DDR5 offers higher speeds (up to 6400 MT/s) and larger capacities.
