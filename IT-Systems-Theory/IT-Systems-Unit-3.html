Algorithm — Concepts & Examples

Definition of Algorithm

An algorithm is a step-by-step, finite set of well-defined instructions to solve a particular problem or perform a specific task. It is written in plain English (or pseudo-code) and is independent of any programming language.

Think of it like a recipe 🍳 — just as a cooking recipe gives you step-by-step instructions to make a dish, an algorithm gives step-by-step instructions for a computer to solve a problem.

🔥 Characteristics of Algorithm (Very Important for Exams)

A good algorithm must have these five characteristics:

#CharacteristicMeaningExample
1FinitenessAlgorithm must terminate (stop) after a finite number of steps. It cannot run forever.After 5 steps, the algorithm gives the answer and stops.
2DefinitenessEach step must be clear and unambiguous. There should be no confusion about what to do."Add A and B" is definite. "Do something with A" is not definite.
3InputAn algorithm should have zero or more inputs given to it before or during execution.Two numbers A and B are inputs to an addition algorithm.
4OutputAn algorithm must produce at least one output (result).The sum of A and B is the output.
5EffectivenessEach step must be basic enough to be carried out. It should be practically doable."Multiply 5 × 3" is effective. "Find all prime numbers" (without specifying limit) is not effective.
Exam Memory Trick

Remember the 5 characteristics as: F-D-I-O-E → Finiteness, Definiteness, Input, Output, Effectiveness. Think: "Five Days In Our Exam" 📝

✅ Advantages of Algorithm

Advantages
  1. Easy to understand: Written in simple language, so anyone can read and follow it.
  2. Step-by-step solution: Breaks a complex problem into small, manageable steps.
  3. Language independent: Not tied to any programming language — can be converted to C, Java, Python, etc.
  4. Easy to debug: Errors can be found easily by checking each step.
  5. Acts as a blueprint: Serves as a plan before actual coding begins.
  6. Easy to communicate: Can be shared with others to explain the logic.

❌ Disadvantages of Algorithm

Disadvantages
  1. Time-consuming: Writing a detailed algorithm takes time, especially for complex problems.
  2. Difficult for complex problems: Very large or complicated problems are hard to express as algorithms.
  3. No standard format: Different people may write the same algorithm in different ways.
  4. Cannot show branching easily: Showing loops and decisions in text form can be confusing compared to flowcharts.
  5. No visual representation: It is purely text-based, which makes it harder to see the overall flow.

📝 How to Write an Algorithm

Rules for Writing
  1. Start with "Step 1: START"
  2. Clearly define all inputs (what data you need)
  3. Write each processing step clearly and in order
  4. Use "IF...THEN...ELSE" for decisions
  5. Use "REPEAT" or "GO TO Step X" for loops
  6. Show the output clearly
  7. End with "Step N: STOP"

⭐ Algorithm Examples (Most Important for Exams)

Example 1: Add Two Numbers

Problem: Write an algorithm to find the sum of two numbers.

Algorithm: Sum of Two Numbers
  1. START
  2. Read two numbers A and B
  3. Calculate SUM = A + B
  4. Display SUM
  5. STOP
Example 2: Check Even or Odd 🔥

Problem: Write an algorithm to check whether a given number is even or odd.

Algorithm: Even or Odd
  1. START
  2. Read a number N
  3. Calculate R = N MOD 2 (remainder when N is divided by 2)
  4. IF R = 0 THEN Display "N is Even"
  5. ELSE Display "N is Odd"
  6. STOP
Example 3: Largest of Three Numbers 🔥

Problem: Write an algorithm to find the largest among three numbers A, B, and C.

Algorithm: Largest of Three Numbers
  1. START
  2. Read three numbers A, B, C
  3. IF A > B AND A > C THEN Display "A is largest"
  4. ELSE IF B > A AND B > C THEN Display "B is largest"
  5. ELSE Display "C is largest"
  6. STOP
Example 4: Factorial of a Number 🔥

Problem: Write an algorithm to find the factorial of a number N. (Factorial of 5 = 5×4×3×2×1 = 120)

Algorithm: Factorial of N
  1. START
  2. Read a number N
  3. Set FACT = 1
  4. Set I = 1
  5. IF I > N THEN go to Step 9
  6. Calculate FACT = FACT × I
  7. Set I = I + 1
  8. Go to Step 5
  9. Display FACT
  10. STOP
Example 5: Swap Two Numbers

Problem: Write an algorithm to swap (exchange) the values of two variables A and B.

Algorithm: Swap Two Numbers
  1. START
  2. Read two numbers A and B
  3. Set TEMP = A
  4. Set A = B
  5. Set B = TEMP
  6. Display A and B (swapped values)
  7. STOP

Here, TEMP is a temporary variable used to hold one value during the swap.

Example 6: Area of a Circle

Problem: Write an algorithm to calculate the area of a circle given its radius.

Algorithm: Area of Circle
  1. START
  2. Read radius R
  3. Calculate AREA = 3.14159 × R × R
  4. Display AREA
  5. STOP

Flowcharts — Symbols, Rules & Examples

Definition of Flowchart

A flowchart is a graphical (diagrammatic) representation of the step-by-step sequence of operations needed to solve a problem. It uses standard symbols connected by arrows (flow lines) to show the flow of logic.

Analogy: If an algorithm is the text version of directions, a flowchart is the map version 🗺️ — both lead you to the destination, but the map is more visual and easier to follow at a glance.

📏 Rules for Drawing a Flowchart

Important Rules
  1. A flowchart must have only one START and one STOP terminal.
  2. The flow should generally go from top to bottom and left to right.
  3. Use standard symbols only — don't invent your own shapes.
  4. All symbols must be connected by flow lines (arrows).
  5. A Decision box must have exactly two exits — one for YES and one for NO.
  6. Write short, clear text inside each symbol.
  7. Use connectors (circles) if flowchart continues on next page.

✅ Advantages of Flowchart

Advantages
  1. Visual representation: Easier to understand at a glance than text.
  2. Better communication: Anyone can follow the logic, even non-programmers.
  3. Easy debugging: Errors in logic can be spotted visually.
  4. Good documentation: Acts as permanent documentation of program logic.
  5. Effective analysis: Helps analyze and understand complex problems.
  6. Coding becomes easier: Once flowchart is ready, writing code is straightforward.

❌ Disadvantages of Flowchart

Disadvantages
  1. Complex for large programs: Flowcharts become very big and messy for complex logic.
  2. Difficult to modify: If logic changes, the entire flowchart may need to be redrawn.
  3. Time-consuming: Drawing a neat flowchart takes significant time.
  4. No standard for detail level: Different people may use different levels of detail.
  5. Cannot show code directly: Flowchart must still be converted to programming code.

HTML5 — Introduction & Basics

What is HTML?

HTML stands for HyperText Markup Language. It is the standard language used to create web pages. It tells the web browser (like Chrome, Firefox) how to structure and display content on a web page.

  • HyperText: Text that contains links to other pages (you can click and jump to another page).
  • Markup: Means you use special "tags" to mark how content should appear.
  • Language: It is not a programming language — it's a markup language (no logic, no calculations).
What is HTML5?

HTML5 is the latest version (5th version) of HTML. It was released in 2014. HTML5 added many new features compared to older versions:

  • New tags for audio, video, canvas (drawing)
  • New form input types (date, email, number, etc.)
  • Better support for mobile devices
  • No need for plugins like Flash to play media

📄 HTML Elements

What is an HTML Element?

An HTML element consists of a start tag, content, and an end tag.

<tagname> Content goes here </tagname>

Example:
<p> This is a paragraph. </p>
 ↑ Start Tag    ↑ Content        ↑ End Tag

The entire thing (start tag + content + end tag) = ONE ELEMENT

Tags are the keywords surrounded by angle brackets < >. They are the building blocks of HTML.

⚙️ HTML Attributes

What are Attributes?

Attributes provide additional information about an HTML element. They are always written in the opening tag and come in name="value" pairs.

<tagname attribute_name="attribute_value"> Content </tagname>

Examples:
<a href="https://google.com"> Click Here </a>
<img src="photo.jpg" width="200" height="150">
<p style="color:red;"> Red text </p>
Common Attributes
  • id — unique identifier for an element
  • class — group identifier (for CSS styling)
  • style — inline CSS styling
  • title — tooltip text on hover
  • src — source file (for images, scripts)
  • href — hyperlink URL (for links)

💬 HTML Comments

Comments are notes written in the code that the browser ignores. They are only visible in the source code, not on the web page. Used to explain code.

The <br> tag inserts a single line break. It is an unpaired (empty) tag — no closing tag needed.

Common Mistake

Pressing Enter in your HTML code does NOT create a new line on the page! The browser ignores extra spaces and line breaks in your source code. You MUST use <br> for a line break.

CSS — Complete Guide

🎨 CSS Introduction

What is CSS?

CSS stands for Cascading Style Sheets. It is used to control the appearance (look and feel) of HTML elements. CSS separates content (HTML) from presentation (styling).

Analogy: If HTML is the skeleton (bones) of a web page, CSS is the skin, clothes, and makeup 💅 — it makes the page look beautiful.

Why Use CSS?
  • Separation of concerns: Keep content (HTML) and design (CSS) separate
  • Consistency: Apply the same style to all pages using one CSS file
  • Easy maintenance: Change look of entire website by editing one CSS file
  • Faster page loading: CSS is cached by the browser
  • Responsive design: Make pages look good on all screen sizes

📝 CSS Syntax 🔥

A CSS rule has three parts: Selector, Property, and Value.

selector {
    property: value;
    property: value;
}

Example:
h1 {
    color: blue;
    font-size: 24px;
    text-align: center;
}

Explanation:
  selector → h1 (which HTML element to style)
  property → color (what to change)
  value    → blue (what to set it to)

PropertyThe style attribute to changecolor, font-size, background
ValueThe value to assign to the propertyred, 20px, #FF0000

🔁 Looping Statements 🔥

Loops allow you to repeat a block of code multiple times.

1. while Loop

Checks condition first, then executes. If condition is false initially, body never runs.

// Print numbers 1 to 5
var i = 1;

while (i <= 5) {
    document.write(i + " ");
    i++;
}
// Output: 1 2 3 4 5

2. do...while Loop

Executes body first, then checks condition. Body always runs at least once.

// Print numbers 1 to 5
var i = 1;

do {
    document.write(i + " ");
    i++;
} while (i <= 5);
// Output: 1 2 3 4 5

// Key difference from while:
var x = 100;
do {
    document.write("Runs once even though x > 5");
} while (x <= 5);
// This STILL runs once! (do executes before checking)

Important Definitions

Algorithm

An algorithm is a finite, step-by-step, well-defined set of instructions to solve a particular problem. It is written in plain English and is independent of any programming language.

Flowchart

A flowchart is a graphical (diagrammatic) representation of the step-by-step sequence of operations needed to solve a problem, using standard symbols connected by flow lines (arrows).

HTML

HTML (HyperText Markup Language) is the standard markup language used to create and structure web pages. It uses tags to define the structure and content of a page.

HTML Element

An HTML element is a complete unit consisting of a start tag, content, and an end tag. Example: <p>Hello</p> is one element.

HTML Attribute

An attribute provides additional information about an HTML element. It is always written in the opening tag as name="value" pairs. Example: href="url", src="image.jpg".

CSS

CSS (Cascading Style Sheets) is a style sheet language used to control the appearance and layout of HTML elements. It separates content from presentation.

CSS Box Model

The CSS Box Model is a concept where every HTML element is treated as a rectangular box with four layers: Content (innermost), Padding, Border, and Margin (outermost).

CSS Selector

A CSS selector is a pattern used to select (target) the HTML element(s) you want to style. Common selectors: element name (h1), class (.classname), id (#idname).

JavaScript

JavaScript is a lightweight, interpreted, client-side scripting language used to make web pages interactive and dynamic. It can modify HTML content, validate forms, and respond to user events.

Variable (in JavaScript)

A variable is a named container used to store data values. In JavaScript, variables are declared using var, let, or const keywords.

Hyperlink

A hyperlink is a clickable text, image, or element on a web page that navigates the user to another page, section, or resource when clicked. Created using the <a> tag.

Web Form

An HTML form is a section of a web page that contains interactive controls (text fields, buttons, checkboxes, etc.) used to collect data from the user and send it to a server for processing.

Part A — Algorithm & Flowcharts
  • ✅ Algorithm = step-by-step textual instructions to solve a problem
  • ✅ 5 Characteristics: Finiteness, Definiteness, Input, Output, Effectiveness (F-D-I-O-E)
  • ✅ Flowchart = graphical representation using standard symbols
  • ✅ 6 Symbols: Oval (Terminal), Rectangle (Process), Diamond (Decision), Parallelogram (I/O), Arrow (Flow), Circle (Connector)
  • ✅ Algorithm is easier to modify; Flowchart is easier to understand
  • ✅ Must know: Add 2 numbers, Even/Odd, Largest of 3, Factorial — both Algorithm + Flowchart
Part B — HTML5
  • ✅ HTML = HyperText Markup Language — creates web page structure
  • ✅ Basic structure: <!DOCTYPE html> → <html> → <head> + <body>
  • ✅ Headings: <h1> to <h6> (h1 = largest)
  • ✅ Formatting: <b>, <i>, <u>, <sub>, <sup>, <del>, <pre>, <mark>
  • ✅ Links: <a href="URL">text</a> — use target="_blank" for new tab
  • ✅ Images: <img src="file" alt="text"> — self-closing tag
  • ✅ Lists: <ol> (numbered), <ul> (bulleted), items in <li>
  • ✅ Tables: <table> → <tr> → <th>/<td>. Know colspan and rowspan!
  • ✅ Colors: By name, Hex (#RRGGBB), RGB (r,g,b)
  • ✅ Forms: text, password, radio, checkbox, select, textarea, submit, reset, file, hidden
  • ✅ GET vs POST: GET = visible URL, less secure | POST = hidden, more secure
  • ✅ Special Tags: <meta> (metadata), <div> (container), <style> (CSS), <iframe> (embed page)
  • ✅ Layouts: Header + Nav + Content + Sidebar + Footer using <div>
Part C — CSS
  • ✅ CSS = Cascading Style Sheets — controls appearance of HTML
  • ✅ Syntax: selector { property: value; }
  • ✅ Selectors: element (p), class (.name), id (#name)
  • ✅ 3 Methods: Inline (style=""), Internal (<style>), External (.css file)
  • ✅ Box Model (MOST IMPORTANT!): Content → Padding → Border → Margin
  • ✅ Position: static, relative, absolute, fixed, sticky
  • ✅ Display: block, inline, inline-block, none
  • ✅ Float: left, right, none. Use clear: both to stop floating.
  • ✅ Link states order: "LoVe HAte" = Link, Visited, Hover, Active
  • ✅ Opacity: 0.0 (invisible) to 1.0 (fully visible)
  • ✅ Class vs Id: Class → multiple elements, dot (.) | Id → unique element, hash (#)
Part D — JavaScript
  • ✅ JavaScript = client-side scripting language for interactivity
  • ✅ Variables: var, let, const. JavaScript is dynamically typed and case-sensitive
  • ✅ Operators: Arithmetic (+,-,*,/,%), Comparison (==,===,!=), Logical (&&, ||, !), Assignment (=,+=)
  • ✅ == vs ===: == checks value only; === checks value AND type
  • ✅ Conditionals: if, if-else, if-else if-else, switch (remember break!)
  • ✅ Loops: while (entry-controlled) vs do...while (exit-controlled, runs at least once)
  • ✅ Output: alert(), document.write(), console.log()
  • ✅ 3 ways to embed JS: Internal (<script>), Event handler (onclick), External (.js file)
