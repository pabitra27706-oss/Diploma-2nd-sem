Algorithm: Definition & Characteristics

Definition

An algorithm is a finite sequence of well-defined, step-by-step instructions to solve a given problem or perform a computation.

Derived from the name of Persian mathematician Al‑Khwarizmi.

Characteristics of an Algorithm
  • Input – zero or more quantities supplied externally.
  • Output – at least one quantity produced.
  • Definiteness – each instruction must be clear and unambiguous.
  • Finiteness – algorithm must terminate after a finite number of steps.
  • Effectiveness – each step must be basic and feasible.

Advantages

  • Systematic problem solving
  • Language independent
  • Easy to understand and debug
  • Can be converted into code easily

Disadvantages

  • Time-consuming to write
  • Difficult for complex logic
  • No standard representation (unlike flowchart)
  • May hide details if too abstract
Example (algorithm: add two numbers)
Step 1: Start
Step 2: Input A, B
Step 3: Sum = A + B
Step 4: Display Sum
Step 5: Stop

Exercise: Write an algorithm to find the largest of three numbers.

Algorithm: Largest of three numbers
1. Start
2. Input three numbers: A, B, C
3. If A > B then
    Max = A
   Else
    Max = B
4. If C > Max then
    Max = C
5. Output "Largest number is: ", Max
6. Stop

This algorithm uses two comparisons and works for any three numbers.

Flowchart: Definition & Overview

What is a Flowchart?

A flowchart is a graphical representation of an algorithm or a process, using standardized symbols to show the sequence of steps and decisions.

It visually depicts the flow of control, making it easier to understand and communicate logic.

Advantages of Flowchart

  • Communication – easy to explain to non‑programmers
  • Effective analysis – detect logical errors early
  • Documentation – serves as program guide
  • Easy to trace and debug

Disadvantages

  • Time‑consuming to draw (especially large programs)
  • Complex logic leads to messy diagrams
  • Modifications may require redrawing
  • No standard for detail level
Tip

Flowcharts are great for small programs or modules. For large systems, use pseudocode alongside.

⬭ (ellipse)TerminatorStart, Stop, or Halt
⬬ (parallelogram)Input / OutputData input (read) or output (print)
⬯ (rectangle)ProcessAny computation or assignment
⬦ (diamond)DecisionBranch (if-else, case)

Note

Some modern tools use slightly different shapes, but the meaning remains consistent. Always include a legend if using non‑standard symbols.

Worked Examples

Example 1: Add two numbers

[Start] → [Input A,B] → [Sum = A+B] → [Print Sum] → [Stop]

Corresponding algorithm is straightforward. This shows the linear flow.

Example 2: Find largest of three numbers
Start → Input A,B,C → if A>B then Max=A else Max=B → if C>Max then Max=C → Output Max → Stop
   [Start]
      ↓
[Input A,B,C]
      ↓
   ◇───Yes───┐
 A>B?         │
   │No        │
   ↓          ↓
Max=B      Max=A
   │          │
   └────┬─────┘
        ↓
    ◇ C>Max? ──No──┐
        │Yes        │
        ↓           │
     Max=C          │
        └─────┬─────┘
              ↓
        [Output Max]
              ↓
            [Stop]
Example 3: Factorial (using loop)

Algorithm (iterative):

fact = 1, i=1; while i ≤ n: fact = fact*i; i=i+1
Flowchart structure

Start → Input n → fact=1, i=1 → ◇ i≤n? (if no → output fact) → (yes) fact=fact*i, i=i+1 → loop back.

Exercise: Draw a flowchart (in words or using symbols) to check whether a number is even or odd.

Even/Odd Checker Algorithm:
1. Start
2. Input number
3. Compute remainder = number % 2
4. If remainder == 0 then
    Print "Even"
   Else
    Print "Odd"
5. Stop

Flowchart: [Start] → [Input num] → ◇ remainder==0? → Yes: [Print Even] → Stop ; No: [Print Odd] → Stop.

Quick recap: Algorithm vs Flowchart

Algorithm = textual, step-by-step. Flowchart = visual, uses symbols. Both have pros/cons; often used together.
