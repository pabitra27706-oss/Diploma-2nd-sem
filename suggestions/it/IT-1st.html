⚡ Binary ↔ Decimal
(1010)₂ = 1×2³ + 0×2² + 1×2¹ + 0×2⁰ = 10
🎯 Gray code (binary → Gray)
MSB same; next bits = XOR(bitᵢ , bitᵢ₋₁)
Ex: 1011 → Gray = 1110
🔁 2’s complement subtraction
A − B = A + (2’s complement of B)
Ignore final carry if present.
📦 BCD
Each decimal digit → 4-bit binary
9 → 1001 , 5 → 0101

📌 Exam pattern alert (2024–2025)  MCQ + explanation mandatory

  • ✅ Each MCQ in Group-A carries 2 marks → 1 mark for correct option + 1 mark for valid explanation.
  • ❌ No marks if explanation is wrong or missing – even if option is correct.
  • 📖 Prioritise understanding “why” behind answers (De Morgan’s working, why Gray code is used, why SSD faster, etc).
  • ⏱️ Subjective part (Group-B): answer any 5 out of 8–10 questions, each 8 marks. Focus on repeated topics: number conversion, 2’s complement, OS types, HTML table, flowchart, malware types, CSS integration.
  • 🎯 Smart revision: practice MCQs with self-explanation; write at least 2–3 flowchart algorithms; practice binary/Gray conversions quickly.
  • 📅 Time management: 2.5 hrs – allocate ~25 min for Group-A (MCQ+explanation), remaining for long answers.
